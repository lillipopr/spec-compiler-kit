# 业务能力检查清单

## 概述

本检查清单用于验证业务能力分析的正确性和完整性。

## 检查清单

### 1. 能力识别

- [ ] 能力使用"动词+名词"格式
- [ ] 能力描述清晰明确
- [ ] 能力可独立交付
- [ ] 能力可验证
- [ ] 能力有业务价值

### 2. 能力分解

- [ ] 能力分解遵循 MECE 原则
- [ ] 能力之间不重叠
- [ ] 能力覆盖所有业务
- [ ] 能力粒度合理
- [ ] 能力可独立演进

### 3. 能力分组

- [ ] 相关能力已分组
- [ ] 组内能力高内聚
- [ ] 组间能力低耦合
- [ ] 分组逻辑清晰
- [ ] 分组数量合理

### 4. 团队对齐

- [ ] 每个能力组对应团队
- [ ] 团队规模合理（3-7 人）
- [ ] 团队职责清晰
- [ ] 沟通路径明确
- [ ] 符合康威定律

### 5. 领域类型

- [ ] 核心域已识别
- [ ] 支撑域已识别
- [ ] 通用域已识别
- [ ] 泛化域已识别
- [ ] 类型判断正确

### 6. 优先级

- [ ] 能力优先级已排序
- [ ] 核心域优先级最高
- [ ] 投入比例合理（80/20 原则）
- [ ] 采购策略明确

### 7. 依赖关系

- [ ] 能力依赖已识别
- [ ] 依赖方向明确
- [ ] 循环依赖已处理
- [ ] 依赖关系可视化
