# 市场分析方法论

## 概述
市场分析是产品决策的基础，通过分析市场规模、用户特征、市场趋势，判断产品机会和可行性。

## 分析框架

```
市场规模 → 用户画像 → 需求分析 → 竞争格局 → 趋势预判 → 机会评估
```

## 1. 市场规模分析

### 1.1 市场规模计算方法

#### TAM-SAM-SOM 模型

```
TAM (Total Addressable Market)
    ↓
SAM (Serviceable Addressable Market)
    ↓
SOM (Serviceable Obtainable Market)
```

- **TAM（总潜在市场）**：理论上可以达到的最大市场规模
- **SAM（可服务市场）**：产品可以覆盖的市场范围
- **SOM（可获得市场）**：短期内可以获得的市场份额

#### 计算示例

```
TAM = 目标用户总数 × 人均年消费额
    = 1亿用户 × 100元/年
    = 100亿元

SAM = TAM × 可覆盖比例
    = 100亿 × 30%（仅覆盖一二线城市）
    = 30亿元

SOM = SAM × 预期市场份额
    = 30亿 × 5%（第一年目标）
    = 1.5亿元
```

### 1.2 市场数据来源

| 数据类型 | 数据来源 | 获取方式 |
|---------|---------|----------|
| 行业规模 | 行业报告 | 艾瑞、易观、IDC |
| 用户规模 | 统计数据 | CNNIC、统计局 |
| 增长趋势 | 历史数据 | 行业历年报告 |
| 细分市场 | 专项调研 | 第三方调研公司 |

### 1.3 市场阶段判断

| 阶段 | 特征 | 策略 |
|-----|------|------|
| **导入期** | 增长缓慢、教育市场 | 培养用户、验证需求 |
| **成长期** | 快速增长、竞争加剧 | 快速扩张、建立壁垒 |
| **成熟期** | 增长放缓、格局稳定 | 精细运营、提升利润 |
| **衰退期** | 增长停滞、需求萎缩 | 转型升级、寻找新机会 |

## 2. 用户画像分析

### 2.1 用户画像要素

```
人口统计 → 行为特征 → 心理特征 → 需求痛点 → 使用场景
```

### 2.2 详细维度

#### 人口统计
- 年龄、性别、地域
- 收入、职业、教育程度
- 家庭状况、婚姻状态

#### 行为特征
- 使用频率、使用时长
- 使用场景、使用设备
- 消费习惯、支付能力

#### 心理特征
- 价值观、生活方式
- 兴趣爱好、个性特点
- 决策风格、风险偏好

#### 需求痛点
- 核心需求是什么
- 现有解决方案的问题
- 未被满足的需求

### 2.3 用户画像模板

```markdown
# 用户画像：职场新人小王

## 基本信息
- 年龄：25岁
- 性别：女
- 地域：上海
- 职业：互联网公司运营
- 收入：月薪8K

## 行为特征
- 每天使用手机5小时以上
- 喜欢尝试新App
- 愿意为效率工具付费
- 对价格敏感度中等

## 心理特征
- 追求效率和品质
- 注重工作与生活平衡
- 喜欢分享和社交
- 对新鲜事物好奇

## 需求痛点
- 工作任务多，经常忘记
- 时间管理困难
- 希望提升工作效率

## 使用场景
- 上下班地铁上查看任务
- 工作间隙记录灵感
- 晚上规划第二天任务
```

### 2.4 用户分层

| 分层方法 | 说明 | 应用 |
|---------|------|------|
| **价值分层** | 按用户价值分高中低 | 资源分配、服务策略 |
| **行为分层** | 按使用行为分活跃度 | 运营策略、召回方案 |
| **生命周期** | 按用户生命周期分 | 留存策略、转化策略 |

## 3. 需求分析

### 3.1 需求类型

| 需求类型 | 定义 | 示例 |
|---------|------|------|
| **基本需求** | 不满足就无法用 | 稳定、安全 |
| **期望需求** | 满足会更满意 | 快速、美观 |
| **兴奋需求** | 超出预期 | 惊喜功能 |
| **反向需求** | 实现反而不满 | 过度打扰 |

### 3.2 需求收集方法

| 方法 | 适用场景 | 优缺点 |
|-----|---------|--------|
| **用户访谈** | 深度了解需求 | 优点：深入；缺点：样本少 |
| **问卷调查** | 大量收集反馈 | 优点：量大；缺点：浅层 |
| **数据分析** | 发现行为模式 | 优点：客观；缺点：无原因 |
| **场景观察** | 了解真实行为 | 优点：真实；缺点：成本高 |
| **竞品分析** | 借鉴成熟方案 | 优点：快速；缺点：非原创 |

### 3.3 Kano 模型

```
          兴奋需求
              ↑
              │ 满意度
基本需求 ──────┼───── 期望需求
              │
              └────────────→ 功能充分度
```

- **基本需求**：必须有，没有就不满意
- **期望需求**：越多越满意
- **兴奋需求**：没有无所谓，有就很惊喜

## 4. 竞争格局分析

### 4.1 竞争格局类型

| 格局 | 特征 | 策略 |
|-----|------|------|
| **垄断** | 一家独大 | 避开正面竞争，寻找细分 |
| **双寡头** | 两强争霸 | 选择站队或差异化 |
| **多头竞争** | 多家混战 | 找准定位，快速突破 |
| **蓝海** | 竞争少 | 抢占先机，建立壁垒 |

### 4.2 竞争强度评估

| 评估维度 | 高竞争 | 低竞争 |
|---------|--------|--------|
| 竞品数量 | >10家 | <3家 |
| 产品同质化 | 高度相似 | 差异明显 |
| 价格战 | 频繁 | 少见 |
| 用户切换成本 | 低 | 高 |
| 进入门槛 | 低 | 高 |

## 5. 趋势预判

### 5.1 趋势类型

| 趋势类型 | 时间跨度 | 预测依据 |
|---------|---------|----------|
| **长期趋势** | 3-5年 | 技术发展、社会变革 |
| **中期趋势** | 1-3年 | 行业演进、政策变化 |
| **短期趋势** | 0.5-1年 | 季节因素、热点事件 |

### 5.2 趋势分析方法

#### PEST 分析
- **P（政治）**：政策法规、政府态度
- **E（经济）**：经济周期、消费能力
- **S（社会）**：人口结构、文化变化
- **T（技术）**：技术革新、产业变革

#### 数据驱动预测
- 历史数据外推
- 增长曲线拟合
- 季节性分析
- 相关因素分析

## 6. 机会评估

### 6.1 机会评估矩阵

```
市场吸引力
    ↑
高 │    ★ 明星机会
   │
中 │    ? 问题机会
   │
低 │    $ 鸡肋机会
   │
   └────────────→ 竞争激烈度
      低  中  高
```

### 6.2 评估标准

| 维度 | 高机会 | 低机会 |
|-----|--------|--------|
| 市场规模 | >100亿 | <10亿 |
| 增长速度 | >30% | <10% |
| 竞争强度 | 低 | 高 |
| 用户痛点 | 强烈 | 一般 |
| 技术可行性 | 高 | 低 |
| 团队匹配度 | 高 | 低 |

### 6.3 决策框架

```
市场机会 × 团队能力 × 资源匹配 = 投资优先级
```

优先级判断：
- 大机会 + 强能力 + 充足资源 = **All in**
- 大机会 + 弱能力 = **补能力或合作**
- 小机会 + 强能力 = **快速验证**
- 小机会 + 弱能力 = **放弃**

## 8. 常见误区

- ❌ 只看市场规模，不看竞争格局
- ❌ 只看用户数量，不看用户质量
- ❌ 只看当前数据，不看趋势变化
- ❌ 只做分析，不做判断
- ❌ 数据来源不权威
