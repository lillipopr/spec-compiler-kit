# 上下文划分检查清单

## 概述

本检查清单用于验证限界上下文划分的正确性和完整性。

## 检查清单

### 1. 边界清晰性

- [ ] 每个上下文都有明确的职责描述
- [ ] 上下文职责单一，不重叠
- [ ] 上下文"不负责"清单已定义
- [ ] 上下文边界可视化（上下文图）
- [ ] 上下文命名清晰、一致

### 2. 语言一致性

- [ ] 上下文内部术语统一（Ubiquitous Language）
- [ ] 核心概念已定义
- [ ] 术语与业务专家一致
- [ ] 代码、文档、讨论使用相同术语
- [ ] 术语表已建立

### 3. 独立性

- [ ] 上下文可独立部署
- [ ] 上下文可独立演进
- [ ] 上下文可独立测试
- [ ] 上下文可独立扩展
- [ ] 上下文变更不影响其他上下文

### 4. 类型识别

- [ ] 上下文类型已正确识别（核心/支撑/通用/泛化）
- [ ] 核心域识别合理
- [ ] 支撑域够用即可
- [ ] 通用域考虑采购
- [ ] 泛化域最小化投入

### 5. 上下文映射

- [ ] 上下游关系已明确
- [ ] 映射模式已选择（O/C, D, ACL, PL, CF 等）
- [ ] 集成方式已确定（同步/异步）
- [ ] 防腐层需求已识别
- [ ] 上下文映射图已绘制

### 6. 团队对齐

- [ ] 一个上下文对应一个团队
- [ ] 团队规模合理（3-7 人）
- [ ] 团队职责与上下文职责匹配
- [ ] 沟通路径清晰
- [ ] 康威定律已考虑

### 7. 数据所有权

- [ ] 每个数据只有一个所有者
- [ ] 数据所有权清晰
- [ ] 数据访问路径明确
- [ ] 数据一致性边界清晰
- [ ] 数据共享规则已定义

## 验证问题

### 边界验证

```
Q1: 这个概念属于哪个上下文？
A: 应该能明确回答，没有歧义

Q2: 两个上下文有重叠吗？
A: 不应该有重叠，如果有需要重新划分

Q3: 上下文可以独立部署吗？
A: 应该可以独立部署
```

### 职责验证

```
Q1: 这个上下文负责什么？
A: 应该有明确的职责描述

Q2: 这个上下文不负责什么？
A: 应该有"不负责"清单

Q3: 职责是否单一？
A: 应该符合单一职责原则
```

### 映射验证

```
Q1: 上下文间有依赖吗？
A: 应该明确依赖关系

Q2: 依赖方向正确吗？
A: 应该从上游到下游

Q3: 集成方式合理吗？
A: 应该根据场景选择合适的集成方式
```

## 常见问题

### 问题 1: 上下文边界模糊

**症状**：不清楚某个概念属于哪个上下文

**解决方案**：
1. 明确各上下文的职责
2. 定义"不负责"清单
3. 使用上下文图可视化

### 问题 2: 上下文职责重叠

**症状**：两个上下文负责相同的事情

**解决方案**：
1. 分析重叠的原因
2. 明确各自职责
3. 考虑合并或重新划分

### 问题 3: 上下文类型错误

**症状**：把通用域当成核心域，浪费资源

**解决方案**：
1. 评估业务价值
2. 评估技术差异
3. 重新识别领域类型
