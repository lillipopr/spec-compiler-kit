## 验收标准

### AC-1: [标准名称] - 正常场景

```gherkin
Given [前置条件]
And [其他前置条件]
When [触发动作]
And [其他动作]
Then [预期结果]
And [其他结果]
```

**示例**：

```gherkin
Given 用户是有效会员（状态=生效中）
And 开通时间超过24小时
When 时间到达每日00:00
Then 用户账户增加100点券
And 用户收到点券到账通知
```

### AC-2: [标准名称] - 边界场景

```gherkin
Given [前置条件]
When [触发动作]
Then [预期结果]
```

**示例**：

```gherkin
Given 用户刚刚开通会员（23:59）
When 当日00:00到达
Then 不发放当日点券
And 从次日开始发放
```

### AC-3: [标准名称] - 异常场景

```gherkin
Given [前置条件]
When [异常触发]
Then [异常处理]
And [后续操作]
```

**示例**：

```gherkin
Given 用户是有效会员
When 点券发放服务故障
Then 记录发放失败日志
And 服务恢复后24小时内补发
And 不影响用户其他功能
```

## 填写指南

### 1. 用户故事三要素

- **用户角色**：具体、明确，不要写"用户"
- **操作**：描述用户行为，不是技术实现
- **价值**：说明为什么需要，用户获得什么价值

### 2. 验收标准

- 使用Given-When-Then格式
- 覆盖正常、边界、异常场景
- 具体可测，不含糊

### 3. INVEST检查

- [ ] **独立**：可独立交付
- [ ] **可协商**：细节可讨论
- [ ] **有价值**：对用户/业务有价值
- [ ] **可估算**：工作量可评估
- [ ] **小**：单次迭代可完成
- [ ] **可测试**：有明确验收标准

### 4. 常见错误

```
❌ 作为用户
✅ 作为会员用户

❌ 我想要一个搜索接口
✅ 我想要搜索商品

❌ 以便提升用户体验
✅ 以便快速找到想要的商品

❌ 应该很快
✅ 3秒内返回结果
```
