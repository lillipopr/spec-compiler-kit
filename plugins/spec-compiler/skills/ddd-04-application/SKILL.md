**必须使用 Read 工具读取以下文件**：

2. 读取文档模板：
   ```

   ```

读取完成后，**必须**输出以下确认信息：

#### 1.3 生成详细文档

## 核心设计原则

### 理论依据

| 原则 | 来源 | 说明 |
|------|------|------|
| 应用服务 | Eric Evans DDD | 用例编排，无业务逻辑 |
| 事务管理 | Martin Fowler | 事务边界管理 |
| 依赖注入 | Martin Fowler | 解耦依赖 |

### 设计质量标准

每个设计元素必须满足：

1. **有理论依据**：能说明为什么这样设计
2. **符合最佳实践**：与业内公认的设计模式一致
3. **可验证**：每个约束可写成 assert，每个用例可转化为测试
4. **可追溯**：设计决策可追溯到 PRD 需求

---

## 设计思路：从领域服务到应用服务

领域专家设计应用服务时，遵循**编排与协调**的原则：

```
领域服务/聚合根（已有能力）
    ↓
用例识别（用户可见的操作）
    ↓
应用服务识别（用例编排）
    ↓
事务边界设计（一致性保证）
    ↓
应用服务设计（接口定义、编排逻辑、异常处理）
    ↓
应用接口用例设计
```
